xeoEngine

A sleek 3D rendering API for WebGL

About

xeoEngine is WebGL-based 3D engine from @xeoLabs

STATUS: It's currently unreleased, while the API is being reviewed.Got some comments or questions about the API? I'd love to have them - email me at:

lindsay.kay AT xeolabs DOT com

Features

  • Models a 3D scene as a fully-editable component-object graph
  • Applies convention over configuration, providing sensible defaults for everything
  • Renders large and complex scenes efficiently
  • Supports a wide range of 3D effects
  • Exports and imports scenes as JSON
  • Has complete API docs

Usage

Simply include xeoengine.min.js in your project to begin creating 3D worlds.

<script src="http://xeoengine.org/build/xeoengine.min.js"></script>

Concept

Create your 3D scene as a component-object graph, as shown below. A scene is essentially a soup of components that are tied together into enties by game objects.

Since xeoEngine provides defaults for everything, we only specify what we need to override, in this case the material and geometry for our game object. The game object will fall back on the scene's default flyweight components (eg. camera, lights etc) for everything else.

var scene = new XEO.Scene();

var material = new XEO.Material(scene, {
    diffuse: [ 0.6, 0.6, 0.7 ]
});

// A 2x2x2 box by default
var geometry = new XEO.Geometry(scene);

var object = new XEO.GameObject(scene, {
    material: myMaterial,
    geometry: myGeometry
});

Animate your scene by setting component properties. All properties fire change events which you can subscribe to.

material.on("diffuse",
     function(value) {
            console.log("Value of material diffuse is now: " + value);
     });

 // This is going to fire our change listener above
 material.diffuse = [0.9, 0.9, 0.6];

Likewise, you can set properties on your scene's default flyweight components:

scene.lights.lights[0].diffuse = [0.5, 1.0, 0.5];

Edit anything in your scene dynamically, at runtime. Create, destroy, link and unlink components, or set any component property.

var texture = new XEO.Texture(scene, {
    src: "myTexture.jpg"
});

material.diffuseMap = texture;

Save and load your scene as JSON:

var json = scene.json;

var scene2 = new XEO.Scene({ json: json });
