xeoEngine

A sleek and powerful 3D engine for WebGL

Download this project as a .zip file Download this project as a tar.gz file

About

Features

  • Scene is defined as an object-component graph
  • API provides sensible defaults for everything
  • Dynamically edit anything in a scene at runtime
  • Save and and load scenes as JSON
  • Optimized to render large scenes efficiently
  • Supports a wide range of 3D effects

Usage

Simply include xeoengine.js in your project to begin creating 3D worlds!

Concept


Plug a 3D scene together as a simple object-component graph, as shown below. xeoEngine provides defaults for everything, so only specify what you need to override. Our example uses all sorts of defaults, such as the camera, lights, even a default canvas.

var scene = new XEO.Scene();

var material = new XEO.Material(scene, {
    diffuse: [ 0.6, 0.6, 0.7 ]
});

// A 2x2x2 box by default
var geometry = new XEO.Geometry(scene);

var object = new XEO.GameObject(scene, {
    material: myMaterial,
    geometry: myGeometry
});

Update scene state by setting component properties. All properties fire change events which you can subscribe to.

material.on("diffuse",
     function(value) {
            console.log("Value of material diffuse is now: " + value);
     });

 // This is going to fire our change listener above
 material.diffuse = [0.9, 0.9, 0.6];

Everything in your scene is dynamically editable. You can create, destroy, link and unlink components, or edit any component property.

  var texture = new XEO.Texture(scene, {
      src: "myTexture.jpg"
  });

  material.diffuseMap = texture;

You can save and load a scene to and from JSON.

 var json = scene.json;

 var scene2 = new XEO.Scene({ json: json });
